## Repository Layout

A PHYTEC BSP workspace has this directory structure:

```
phytec-bsp-workspace/          ← BSPDIR
├── sources/                   ← all layer sources
│   ├── poky/                  ← Yocto Project (meta, meta-poky)
│   ├── meta-openembedded/     ← OpenEmbedded layer collection
│   ├── meta-imx/              ← NXP i.MX BSP layers
│   ├── meta-phytec/           ← PHYTEC software recipes
│   ├── meta-phytec-bsp/       ← PHYTEC hardware BSP
│   └── meta-security/         ← Security framework layers
│
├── keys/                      ← Signing keys (NOT in git)
│   └── fit/
│       ├── fit-signing-key.pem  ← Private key (offline only)
│       └── fit-signing-key.crt  ← Public certificate
│
└── build/                     ← Build directory (created by setup script)
    ├── conf/
    │   ├── bblayers.conf      ← Layer list
    │   └── local.conf         ← Per-build configuration
    ├── tmp/                   ← Build temporaries (generated)
    └── downloads/             ← Source tarballs (shared cache)
```

## bblayers.conf

The `bblayers.conf` file lists every layer that BitBake will process. Layer order here does not determine priority — priority is set per-layer in `conf/layer.conf` via `BBFILE_PRIORITY`. However, listing order affects how BitBake resolves certain path-based configurations.

```bitbake
# /home/user/phytec-bsp-workspace/build/conf/bblayers.conf
# Generated by phytec-setup-release.sh, manually tuned for secure boot

BSPDIR := "${@os.path.abspath(os.path.dirname(d.getVar('FILE')) + '/../../')}"

BBLAYERS ?= " \
  ${BSPDIR}/sources/poky/meta \
  ${BSPDIR}/sources/poky/meta-poky \
  ${BSPDIR}/sources/meta-openembedded/meta-oe \
  ${BSPDIR}/sources/meta-openembedded/meta-python \
  ${BSPDIR}/sources/meta-openembedded/meta-networking \
  ${BSPDIR}/sources/meta-openembedded/meta-filesystems \
  ${BSPDIR}/sources/meta-security \
  ${BSPDIR}/sources/meta-security/meta-tpm \
  ${BSPDIR}/sources/meta-imx/meta-bsp \
  ${BSPDIR}/sources/meta-imx/meta-sdk \
  ${BSPDIR}/sources/meta-phytec \
  ${BSPDIR}/sources/meta-phytec-bsp \
  "
```

### Validating bblayers.conf

After editing `bblayers.conf`, verify it is syntactically correct and all layers are found:

```bash
bitbake-layers show-layers
```

Expected output:

```
layer                 path                                      priority
==========================================================================
meta                  .../sources/poky/meta                              5
meta-poky             .../sources/poky/meta-poky                         5
meta-oe               .../sources/meta-openembedded/meta-oe              6
```

### Verifying Layer Compatibility


## local.conf: Complete Production Configuration

This is the authoritative `local.conf` for a secure-boot-enabled phyCORE-i.MX8MP build. Each section is annotated.

```bitbake
# ═══════════════════════════════════════════════════════════════════
# MACHINE AND DISTRO SELECTION
# ═══════════════════════════════════════════════════════════════════

# Target machine: phyBOARD-Pollux i.MX8MP 3GB LPDDR4
MACHINE = "phyboard-pollux-imx8mp-3"

# PHYTEC headless distribution (no graphical stack)
# Alternative: "phytec-securiphy-distro" when using full securiphy
DISTRO = "phytec-headless-distro"

# ═══════════════════════════════════════════════════════════════════
# BUILD PERFORMANCE TUNING
# ═══════════════════════════════════════════════════════════════════

# Parallel task execution (set to number of CPU threads)
BB_NUMBER_THREADS ?= "8"

# Parallel make jobs within each task
PARALLEL_MAKE ?= "-j8"

# Shared state cache (speeds up rebuilds across workspaces)
SSTATE_DIR ?= "${BSPDIR}/sstate-cache"

# Downloaded source archive cache
DL_DIR ?= "${BSPDIR}/downloads"

# Temporary build directory (large: 30–100 GB depending on build scope)
# TMPDIR = "${BSPDIR}/tmp"  # Default: ${TOPDIR}/tmp

# ═══════════════════════════════════════════════════════════════════
# KERNEL CONFIGURATION
# ═══════════════════════════════════════════════════════════════════

# Produce a FIT image instead of raw kernel Image
KERNEL_IMAGETYPE = "fitImage"

# Add kernel-fitimage.bbclass to kernel recipe
KERNEL_CLASSES = "kernel-fitimage"

# Use NXP downstream kernel
PREFERRED_PROVIDER_virtual/kernel = "linux-imx"

# Pin kernel version (% is wildcard matching micro versions)
PREFERRED_VERSION_linux-imx = "6.6%"

# ═══════════════════════════════════════════════════════════════════
# FIT IMAGE SIGNING CONFIGURATION
# ═══════════════════════════════════════════════════════════════════

# Enable FIT signing (set to "0" to disable for initial development)
UBOOT_SIGN_ENABLE = "1"

# Directory containing signing key files
# Must contain: fit-signing-key.pem (private key)
#               fit-signing-key.crt (X.509 certificate with public key)
# IMPORTANT: This path must be absolute or anchored to TOPDIR
# TOPDIR = the build/ directory
UBOOT_SIGN_KEYDIR = "${TOPDIR}/../../keys/fit"

# Key pair base name (no file extension)
# Matches: ${UBOOT_SIGN_KEYDIR}/fit-signing-key.pem
#          ${UBOOT_SIGN_KEYDIR}/fit-signing-key.crt
UBOOT_SIGN_KEYNAME = "fit-signing-key"

# Do NOT auto-generate keys during build
# Production policy: keys exist before build starts
FIT_GENERATE_KEYS = "0"

# RSA-2048 with SHA-256 (compatible with all U-Boot RSA configurations)
FIT_SIGN_ALG = "rsa2048"
FIT_HASH_ALG = "sha256"
FIT_SIGN_NUMBITS = "2048"

# mkimage -f options: reserve space for signature data in DTB
# -p 2000: 2000-byte pad ensures signature fits without DTB resize
UBOOT_MKIMAGE_DTCOPTS = "-I dts -O dtb -p 2000"

# Sign each image node individually in addition to configuration node
# Useful for debugging; may not be required for standard verified boot
# FIT_SIGN_INDIVIDUAL = "0"

# ═══════════════════════════════════════════════════════════════════
# U-BOOT CONFIGURATION
# ═══════════════════════════════════════════════════════════════════

# Use NXP downstream U-Boot
PREFERRED_PROVIDER_virtual/bootloader = "u-boot-imx"

# U-Boot defconfig matching phyCORE-i.MX8MP
UBOOT_MACHINE = "phycore-imx8mp_defconfig"

# U-Boot DTB (receives embedded FIT public key after signing)
UBOOT_DTB_BINARY = "u-boot.dtb"

# ═══════════════════════════════════════════════════════════════════
# TF-A AND OP-TEE
# ═══════════════════════════════════════════════════════════════════

# Use NXP downstream TF-A
PREFERRED_PROVIDER_virtual/trusted-firmware-a = "trusted-firmware-a-imx"

# Enable OP-TEE integration
MACHINE_FEATURES:append = " optee"

# OP-TEE memory configuration (values from PHYTEC SOM reference)
# These are typically set in the machine conf; shown here for reference:
# OPTEE_TARGET = "plat-imx"
# OPTEE_PLATFORM = "mx8mpevk"

# ═══════════════════════════════════════════════════════════════════
# IMAGE CONFIGURATION
# ═══════════════════════════════════════════════════════════════════

# Output formats:
# wic.gz: compressed disk image (write with bmaptool or dd | gunzip)
# wic.bmap: block map for efficient sparse-image flashing
# ext4: raw filesystem (input for dm-verity hash generation)
IMAGE_FSTYPES = "wic.gz wic.bmap ext4"

# Disk layout: boot partition + rootfs partition
WKS_FILE = "imx-imx-boot-bootpart.wks.in"

# ═══════════════════════════════════════════════════════════════════
# SECURITY HARDENING
# ═══════════════════════════════════════════════════════════════════

# Enable security meta-features
DISTRO_FEATURES:append = " security"

# IMA: Integrity Measurement Architecture (optional; see ima.bbclass)
# DISTRO_FEATURES:append = " ima"
# Enabling IMA adds kernel command-line IMA parameters and policy files.
# Only enable if you have an IMA policy ready.

# Read-only root filesystem (required for dm-verity)
IMAGE_FEATURES = "read-only-rootfs"

# CRITICAL: Remove debug convenience features from production
IMAGE_FEATURES:remove = "debug-tweaks"
EXTRA_IMAGE_FEATURES = ""

# Disable SSH root login (default in most distros; explicit here)
# IMAGE_FEATURES:remove = "allow-root-login"

# ═══════════════════════════════════════════════════════════════════
# PACKAGE EXCLUSIONS (PRODUCTION)
# ═══════════════════════════════════════════════════════════════════

# Remove compiler toolchain (prevents in-field compilation)
PACKAGE_EXCLUDE = "gcc g++ gdb binutils"

# Remove debug/trace tools
PACKAGE_EXCLUDE:append = " strace ltrace valgrind"

# Remove package manager (prevents in-field package changes)
# Uncomment if using opkg-based image:
# PACKAGE_EXCLUDE:append = " opkg"
```

## Key File Preparation

Before invoking `bitbake`, FIT signing keys must exist at `UBOOT_SIGN_KEYDIR`. The build will fail with a cryptic error if keys are missing.

### Generate FIT Signing Key Pair

```bash
# Create key directory (one level above build/)
mkdir -p /home/user/phytec-bsp-workspace/keys/fit
cd /home/user/phytec-bsp-workspace/keys/fit

# Generate RSA-2048 private key
openssl genrsa -out fit-signing-key.pem 2048

# Verify key was created
openssl rsa -in fit-signing-key.pem -noout -text | head -5
# Should show: Private-Key: (2048 bit)

# Generate X.509 certificate (self-signed)
# The CN field appears in U-Boot log when loading FIT
openssl req -new -x509 \
    -key fit-signing-key.pem \
    -out fit-signing-key.crt \
    -days 7300 \
    -subj "/C=DE/O=MyCompany/CN=FIT Signing Key/OU=Production"

# Verify certificate
openssl x509 -in fit-signing-key.crt -noout -text | \
    grep -E "Subject:|Not After|Public-Key:"
```

### Key File Permissions

```bash
# Private key: readable only by build user
chmod 600 /home/user/phytec-bsp-workspace/keys/fit/fit-signing-key.pem

# Certificate: readable by build process
chmod 644 /home/user/phytec-bsp-workspace/keys/fit/fit-signing-key.crt

# Verify
ls -la /home/user/phytec-bsp-workspace/keys/fit/
# -rw------- fit-signing-key.pem
# -rw-r--r-- fit-signing-key.crt
```

### Verify UBOOT_SIGN_KEYDIR Resolution

Before building, verify that BitBake resolves the key path correctly:

```bash
cd /home/user/phytec-bsp-workspace/build

bitbake -e virtual/kernel | grep -E "^UBOOT_SIGN_KEYDIR=|^UBOOT_SIGN_KEYNAME="
# UBOOT_SIGN_KEYDIR="/home/user/phytec-bsp-workspace/keys/fit"
# UBOOT_SIGN_KEYNAME="fit-signing-key"

# Verify the resolved path contains the key files:
KEY_DIR=$(bitbake -e virtual/kernel | grep "^UBOOT_SIGN_KEYDIR=" | cut -d'"' -f2)
ls -la "${KEY_DIR}/"
```

## Build Commands

### Development Build (Signing Disabled)

For initial bringup before keys are generated:

```bash
# In local.conf: set UBOOT_SIGN_ENABLE = "0"
# Then build:
bitbake phytec-headless-image

# Verify unsigned fitImage was produced:
ls -la tmp/deploy/images/phyboard-pollux-imx8mp-3/fitImage
dumpimage -l tmp/deploy/images/phyboard-pollux-imx8mp-3/fitImage
# Should show kernel/dtb nodes but NO "Sign algo" line
```

### Production Build (Signing Enabled)

```bash
# Ensure keys are in place
ls /home/user/phytec-bsp-workspace/keys/fit/

# Build with signing enabled (UBOOT_SIGN_ENABLE = "1" in local.conf)
bitbake phytec-securiphy-image

# Build just the FIT image (faster iteration):
bitbake virtual/kernel

# Build U-Boot with embedded public key:
bitbake virtual/bootloader

# Verify signed FIT image:
ls -la tmp/deploy/images/phyboard-pollux-imx8mp-3/fitImage
dumpimage -l tmp/deploy/images/phyboard-pollux-imx8mp-3/fitImage
# Should show: "Sign algo: sha256,rsa2048:fit-signing-key"
```

### Verifying the Complete Build

```bash
DEPLOY=tmp/deploy/images/phyboard-pollux-imx8mp-3

# 1. Verify FIT image structure and signature nodes
dumpimage -l ${DEPLOY}/fitImage
# Look for: "Sign algo: sha256,rsa2048:fit-signing-key" in configuration section

# 2. Verify U-Boot DTB contains embedded public key
fdtdump ${DEPLOY}/u-boot.dtb 2>/dev/null | grep -A 10 "signature"
# Should show: "fit-signing-key" key node with rsa-modulus, rsa-exponent

# 3. Verify imx-boot image was produced
ls -la ${DEPLOY}/imx-boot-phyboard-pollux-imx8mp-3-sd.bin-flash_evk

# 4. Check FIT SHA256 consistency
sha256sum ${DEPLOY}/fitImage
```

## Common Configuration Errors

### Error: "FIT image signing key not found"

**Symptom:** BitBake task `do_uboot_assemble_fitimage` fails with:
```
Error: key file .../fit-signing-key.key not found
```

**Cause:** `mkimage` looks for a `.key` extension but the private key file uses `.pem` extension.

**Fix:** Create a symlink or copy:
```bash
cd /home/user/phytec-bsp-workspace/keys/fit
cp fit-signing-key.pem fit-signing-key.key
# Or: ln -s fit-signing-key.pem fit-signing-key.key
```

Note: Some versions of `mkimage` require `.key`; others accept `.pem`. Check your U-Boot/mkimage version and the error message carefully.

### Error: "DTB padding too small for signature"

**Symptom:** `do_uboot_assemble_fitimage` fails with:
```
FDT: Failed to add FDT stacked overlays
```
or the FIT image is produced but `verified boot` fails at runtime.

**Cause:** `UBOOT_MKIMAGE_DTCOPTS = "-I dts -O dtb -p 2000"` padding value is too small for the signature data (RSA-2048 = 256 bytes of signature, plus public key modulus, exponent, etc.).

**Fix:** Increase the padding value:
```bitbake
UBOOT_MKIMAGE_DTCOPTS = "-I dts -O dtb -p 4096"
```

### Error: "UBOOT_SIGN_KEYDIR not set"

**Symptom:** U-Boot DTB does not contain public key node. FIT verification fails at runtime with "RSA: key not found."

**Cause:** `UBOOT_SIGN_KEYDIR` is empty or resolves to a non-existent directory.

**Diagnosis:**
```bash
bitbake -e virtual/bootloader | grep "^UBOOT_SIGN_KEYDIR"
ls -la "$(bitbake -e virtual/bootloader | grep "^UBOOT_SIGN_KEYDIR" | cut -d'"' -f2)"
```

**Fix:** Ensure the variable is set in `local.conf` and the directory exists with correct key files.

### Error: Signing with UBOOT_SIGN_ENABLE=1 produces unsigned fitImage

**Symptom:** `dumpimage -l fitImage` shows no `Sign algo` line.

**Cause:** `UBOOT_SIGN_ENABLE = "1"` is set but `KERNEL_CLASSES = "kernel-fitimage"` is missing.

**Fix:** Both variables must be set:
```bitbake
KERNEL_IMAGETYPE = "fitImage"
KERNEL_CLASSES = "kernel-fitimage"
UBOOT_SIGN_ENABLE = "1"
```
